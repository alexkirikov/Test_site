# Autonomy Planner — багатосторінковий сайт

Сайт розбито на окремі HTML-сторінки зі спільними, підключеними CSS та скриптами.

## Структура

```
site/
├── index.html          Головна
├── features.html       Можливості продукту
├── how.html            Як це працює
├── homeowners.html     Для власників будинків
├── installers.html     Для інсталяторів / партнерів
├── pricing.html        Ціни / тарифи + FAQ
├── cases.html          Кейси
├── about.html          Про нас
├── blog.html           Блог / База знань
├── article.html        Сторінка статті
├── contact.html        Контакти / заявка
├── app.html            Інтерактивний планувальник (демо)
│
├── css/
│   ├── styles.css      Точка входу — підключає всі токени
│   └── tokens/         Кольори, типографіка, відступи, радіуси, рух, шрифти
│
├── js/
│   ├── ds-bundle.js    Скомпільовані компоненти дизайн-системи
│   ├── logo-data.js    Логотип (data-URI)
│   ├── Shared.jsx      Спільні хелпери (Icon, Section, Placeholder…)
│   ├── Hero.jsx        Навбар + герой головної
│   ├── Value.jsx       Секції цінності головної
│   ├── Proof.jsx       Відгуки + B2C/B2B таби
│   ├── Pricing.jsx     Тарифи + CTA + футер
│   ├── PagesProduct.jsx, PagesBiz.jsx, PagesContent.jsx
│   ├── PlannerApp.jsx  Демо-планувальник
│   └── site.jsx        Маршрути + збірка головної + монтаж сторінки
│
└── assets/
    └── ap-logo.png     Логотип
```

## Як працює

- Кожна сторінка підключає **один CSS** (`css/styles.css`) і набір **спільних JS-файлів**.
  Браузер кешує спільні скрипти між сторінками.
- Активну сторінку задає рядок у `<head>`:
  `window.AP_PAGE = { name: 'FeaturesPage', active: 'features' };`
  Файл `js/site.jsx` зчитує його й монтує потрібний компонент (з навбаром, окрім `app.html`).
- Навігація між сторінками — реальні переходи: `onNav('pricing')` → `pricing.html`
  (карта маршрутів — у `js/site.jsx`, `window.AP_ROUTES`).

## Запуск

Компоненти написані як JSX і компілюються в браузері (Babel), тому сторінки треба
**відкривати через локальний сервер**, а не подвійним кліком (`file://` блокує
завантаження `.jsx`). Наприклад, з папки `site/`:

```
python3 -m http.server 8000
```

і відкрити `http://localhost:8000/`.

> Для офлайн-версії в один файл скористайтесь `export/Autonomy Planner — сайт.html`.
